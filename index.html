PidiLang

The Knowingly Adaptive, Highly Expressive Programming Language

1. Project Philosophy

PidiLang (Pidgin Language) is built on the principle that programming should be universally accessible and intuitive. By leveraging the expressive power and accessibility of West African Pidgin English, PidiLang aims to lower the barrier to entry for new developers and bring a dynamic, culturally rich voice to the coding world. Our mission is to make development feel less like a chore and more like a conversation.

2. Core Lexicon and Keywords

The language is built around a small, powerful set of commands, rooted in everyday communication.

Category PidiLang Keyword Function
Data/Variables DAT_ME Declaration and Assignment
Data Types PLENTY_THINGS Defines a List/Array
Data Types CONTAINER Defines an Object/Dictionary
Functions DO_WORK Function Definition
Functions GIVE_BACK Return a value
Output TALK_AM Print to console
Conditional IF_E_TRUE / ELSE_IF_NO If / Else Conditionals
Looping WAKA_ROUND / WHILE_E_DEY For Loop / While Loop
Exit / Success IF_NO_WAHALA Graceful Return/Exit

3. Error Handling and Exception Management

We manage errors, or "wahala," using three primary commands:

TRY_YOUR_BEST

Marks the block of code where unexpected issues might arise. Essentially the try block.

IF_PROBLEM_DEY

The error catcher. Code inside this block runs only if a wahala is detected in the TRY_YOUR_BEST block. This is the catch block.

FINALLY_GREET

Code that executes unconditionally, whether a wahala occurred or not. The finally block.

Example: Handling Wahala

TRY_YOUR_BEST (
    DAT_ME num1 = 10
    DAT_ME num2 = 0
    DAT_ME result = num1 / num2 // Wahala go dey here!
    TALK_AM "Result is: " + result
)

IF_PROBLEM_DEY (
    TALK_AM "E be like say math no straight. No divide by zero!"
)

FINALLY_GREET (
    TALK_AM "Work don finish, we thank God."
)
