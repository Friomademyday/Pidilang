1. Project Philosophy and Intent

PidiLang is an open-source, highly expressive programming language built on the West African Pidgin English dialect. Our core mission is to **democratize access to coding** by utilizing an intuitive, globally understood, and culturally rich lexicon, making the journey from novice to competent developer as straightforward as a conversation.

2. Installation and Setup

To get started with the PidiLang interpreter (PidiShell), you must first download the CLI tool:

$ install pidilang --global

Verify your installation:

$ pidishell --version
PidiLang Runtime v1.0.0 (Wahala-Free)

3. Core Lexicon and Keywords

PidiLang is defined by its concise and powerful set of commands:

Category PidiLang Keyword Programming Function / Intent
Data/Vars DAT_ME Variable declaration and assignment (`let`, `var`).
Collection PLENTY_THINGS / CONTAINER Defines an Array/List or an Object/Dictionary, respectively.
Control Flow IF_E_TRUE / ELSE_IF_NO Standard Conditional (`if`/`else`).
Iteration WAKA_ROUND Iterates over elements in a collection (`for...in`).
Functions DO_WORK / GIVE_BACK Function definition and returning a value (`return`).
I/O & Exit TALK_AM / IF_NO_WAHALA Prints output / Graceful program exit.

4. Exception Management (Wahala Handling)

PidiLang uses clear, direct blocks to manage runtime errors, or **"Wahala"** (Trouble).

TRY_YOUR_BEST

The block of code where exceptions are anticipated. The equivalent of a standard `try` block.

IF_PROBLEM_DEY

The handler for catching and mitigating the `Wahala`. Executes only upon failure (`catch`).

FINALLY_GREET

The guaranteed cleanup block. Runs regardless of success or failure (`finally`).

Example: Division by Zero Wahala

TRY_YOUR_BEST (
    DAT_ME dividend = 100
    DAT_ME divisor = 0
    DAT_ME result = dividend / divisor  // Fatal operation expected here
    TALK_AM "Result is: " + result
)

IF_PROBLEM_DEY (
    TALK_AM "E be like say math no straight. No divide by zero, abi?"
)

FINALLY_GREET (
    TALK_AM "Session complete. Work don finish."
)

5. Roadmap and Development

We are committed to continuous integration and expansion of the PidiLang ecosystem. Key upcoming features include:

  • **v1.1:** Native support for asynchronous operations (`WAIT_SMALL`).
  • **v1.2:** Compiler integration for faster binary execution.
  • **v1.3:** Pidi Package Manager (`pidi-pkg`) for dependency management.
